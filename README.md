# company-classification-analysis

EDINETから取得した有価証券報告書の【事業内容】テキストをもとに、SBERT(Sentence-BERT)による自然言語処理で、企業を東証33業種へ確率的に分類・分析するプロジェクト。「業種コード」という離散ラベルだけでは見えない、企業の事業内容の類似度や立ち位置を、埋め込みベクトルという連続空間上で可視化する。

## パイプライン全体像

```
EDINET API → XBRL解析 → テキストクレンジング
    → [TF-IDFベースライン] / [SBERT埋め込み → ファインチューニング]
    → 業種分類・評価 → UMAP+KMeansによる可視化
```

対象企業数: 3,637社(上場企業)、業種ラベル: 東証33業種区分。

---

## 1. データ収集(EDINETスクレイピング)

`src/scraper/edinet.py`

- EDINET API(v2)から、指定期間(2025年4月〜2026年3月)の有価証券報告書一覧を日次で取得し、ZIP形式で保存。

`src/processor/edinet_process.py`

- 保存したZIP内のXBRLファイルから、BeautifulSoupで【事業の内容】【業績等の概要】に相当するセクションをテキスト抽出し、CSV化(`edinet_analysis_data.csv`)。

---

## 2. データクレンジング

`notebooks/clense_data.ipynb`

- 抽出したbusiness_descriptionの欠損値を確認。欠損していた企業(Zenken株式会社、株式会社フジクラなど)についてテキストを個別に補完し、残る欠損行は削除。
- 最終的なクレンジング済みデータを`edinet_analysis_data_clns.csv`として保存。

---

## 3. TF-IDFベースのベースライン分析

`notebooks/tfidf.ipynb`

- SudachiPyで事業内容テキストを形態素解析し、名詞のみを抽出(`src/processor/word_process.py`)。
- 全企業で77,651種類のユニークな名詞、延べ1,457,988語を確認。
- TF-IDFベクトル化(3,637社 × 12,097語の疎行列)を実施し、コサイン類似度で企業間の類似度を計算。
- 具体例として、株式会社PALTAC(化粧品・日用品卸)に類似する企業として資生堂・コーセーホールディングス等のドラッグストア・卸売関連企業が上位に並ぶことを確認。
- UMAP(4次元)+KMeans(33クラスタ)による可視化、および企業名同士の「足し算・引き算」(`corporate_arithmetic`)による類似企業探索を実装(例: サンリオ+東宝 → 松竹・東映など映像コンテンツ企業が上位)。

---

## 4. SBERTによるベクトル化

`notebooks/vectorize_category_sbert.ipynb`、`notebooks/vectorize_company_sbert.ipynb`

- 多言語対応の事前学習済みモデル`intfloat/multilingual-e5-base`を使用。
- 33業種の業種説明文、および3,637社の事業内容テキスト(チャンク分割後12,442チャンク)を、それぞれ768次元ベクトルに変換。
- チャンク単位でベクトル化した後、企業・業種ごとにベクトルを平均化して集約。
- ファインチューニング前・後のモデル両方でベクトル化を実施し、比較できる形で`models/`ディレクトリに保存(`.npy`及び`.pkl`)。

---

## 5. ファインチューニング

`notebooks/execute_fine_tuning.ipynb`

- `multilingual-e5-base`に分類ヘッドを追加し、業種ラベル(33クラス)を教師信号として分類タスクでファインチューニング。
- 訓練・検証データを8:2に分割(層化抽出)。3エポック学習し、training_loss ≈ 1.56まで低下。
- 学習済みモデルを`models/e5_edinet_finetuned`に保存。

---

## 6. 分類評価

`notebooks/company_classifier.ipynb`

- ファインチューニング後の企業ベクトル(3,637×768)と業種説明ベクトル(33×768)の内積により、企業ごとに33業種への「スコア」を算出(3,637×33の行列)。
- 正解業種の順位・スコアを個社ごとに検証。
- 具体例: KDDI株式会社の事業内容テキストから、実際の業種(情報・通信業)が第1位で的中。
- 定量評価: Top-1精度 75.2%、Top-3精度 90.8%、Top-5精度 95.4%。
- 保存したファインチューニング済み分類モデルに、任意の新規テキスト(例: 自動車部品メーカーの事業内容)を入力し、業種を予測するデモも実装(正しく「輸送用機器」と予測)。

---

## 7. 可視化

`notebooks/visualize_umap.ipynb`、`notebooks/visualize_kmeans.ipynb`

- ファインチューニング前後の企業埋め込み(768次元)を、UMAPで3〜4次元に圧縮し、Plotlyで3D散布図として可視化。
- KMeans(k=33)によるクラスタリング結果を、UMAP空間上に重ねて表示。
- 生成された画像は`img/`に保存(`tf-idf_umap.png`、`sbert_umap.png`、`sbert_ft_umap.png`、`kmeans.png`、`kmeans_ft.png`、`rank_business_noun.png`)。

---

## 8. 応用: ベクトル演算

`notebooks/use_vector.ipynb`、`src/evaluator/word_vector.py`

- `quick_search`: 自由記述テキストから、意味的に近い企業を検索する意味検索機能。
- `concept_arithmetic`: 企業ベクトルに対する概念の加算・減算(例: 「KDDI − キャラクター + 不動産」で通信・不動産関連の類似企業を抽出)。word2vecの「king - man + woman = queen」的な演算を、企業の埋め込みベクトル上で試みた応用例。

## 今後の課題

- **クレンジング処理の自動化**: 欠損値の補完が一部手作業になっている箇所を、ルールベースまたは別途スクレイピングによる再取得で自動化する余地がある。
- **クラスタリング結果の定量評価**: UMAP+KMeansによる可視化は説得力があるが、クラスタと実際の業種ラベルの一致度(ARIやNMIなどの指標)による定量評価が未実施。
- **外れ値企業の調査**: 分類評価で正解業種の順位が著しく低かった企業(空運業に分類されるべき企業が30位相当など)について、事業内容テキスト自体の質や複数事業セグメントを持つ企業特有の課題を個別に調査する余地がある。
